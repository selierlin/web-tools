# 修改工具技能

## 技能描述
协助用户修改和优化现有的工具页面。

## 使用场景
- 修复工具中的 Bug
- 添加新功能到现有工具
- 优化工具性能和用户体验
- 统一工具样式

## 执行步骤

### 1. 分析现有代码
- 阅读工具页面的 HTML/CSS/JS
- 理解当前功能实现
- 识别需要修改的部分

### 2. 制定修改方案
- 明确修改目标
- 评估影响范围
- 确定最佳实现方式

### 3. 执行修改
- 保持代码风格一致
- 遵循项目规范
- 添加必要的注释

### 4. 验证修改
- 检查功能是否正常
- 确保没有引入新问题
- 测试边界情况

## 常见修改类型

### 样式调整
```css
/* 修改主题色 */
:root {
    --primary-color: #新颜色;
}

/* 调整布局 */
.container {
    max-width: 新宽度;
}
```

### 功能增强
```javascript
// 添加新功能
function newFeature() {
    // 实现代码
}

// 绑定事件
document.getElementById('btn').addEventListener('click', newFeature);
```

### Bug 修复
```javascript
// 添加错误处理
try {
    // 可能出错的代码
} catch (error) {
    console.error('操作失败:', error);
    alert('操作失败，请检查输入');
}
```

## 注意事项
- 修改前备份原文件
- 小步修改，频繁测试
- 保持向后兼容
- 更新相关注释
